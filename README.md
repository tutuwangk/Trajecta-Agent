# Trajecta-Agent

把零散旅行资料和想法整理好，给你一份每日行程规划。

Trajecta-Agent 面向真实旅行决策场景。用户输入目的地、天数、交通偏好、路线目标和行程强度等信息，并粘贴攻略、地点清单、餐厅推荐或自由描述后，系统会整理地点、确认位置，并生成可继续调整的每日路线。


## 功能简介

- 收集目的地、天数、酒店名、出行人数、预算、兴趣偏好、交通偏好、路线目标和行程强度
- 识别资料里的景点、餐厅、酒店和其他地点
- 生成轻量地点池，让用户用少量操作完成关键决策
- 生成按天拆分的旅行路线
- 在结果页继续修改，让 Agent 重新整理

适合的输入包括攻略摘录、餐厅清单、朋友推荐、酒店地址、备忘录，以及自然语言形式的旅行想法。

## Agent 工作流程


1. 收集行程约束  
   前端同时提交结构化行程设置与原始资料。结构化部分包括目的地、天数、酒店名、出行人数、预算、兴趣偏好、交通偏好、路线目标和行程强度；原始资料部分承载攻略、地点线索、餐厅推荐和补充说明。两类输入共同构成规划约束。

2. 解析资料与用户意图  
   后端从原始资料中抽取候选地点、区域、餐厅、酒店、交通节点以及用户意图，例如“必去”“可去可不去”“晚上想看夜景”等。这些信息会与结构化字段合并，用于后续地点判断和路线取舍。

3. 完成地图匹配与地点整理  
   系统调用高德服务，将候选地点尽量落到真实地图对象上，并结合来源证据、用户表达、地点类型、酒店位置、路线目标和兴趣偏好，为每个地点生成默认决策。整理后的地点池保留“必去、待定、移除、改名”等轻量操作，用户可在关键处补充判断。

4. 在约束框架下生成路线  
   进入正式规划的地点，需要同时满足地图确认和最终决策要求。规划阶段会综合考虑以下因素：
   - 天数与每日时间边界
   - 酒店作为每日起终点的参考位置
   - 交通偏好对应的移动成本
   - 路线目标对应的排序倾向
   - 兴趣偏好与地点权重
   - 行程强度上限


5. 校验并修正初版路线  
   初版路线生成后，后端会继续检查地点冲突、绕路、节奏过密、饭点安排、未匹配地点和约束偏离情况，并对路线做一轮修正，再输出路线概览、每日路线、未安排原因和风险提示。

6. 根据反馈继续重排  
   用户可以通过快捷操作或自然语言继续提出修改，例如调整某一天的节奏、更换酒店、后置某个地点或重新安排顺路顺序。系统会基于当前地点池、约束条件和既有路线结果进行局部重排。

## 本地部署

需要本地安装：

- Python 3
- Node.js
- pnpm

复制 `.env.example` 为 `.env`，填入真实密钥：

后端会自动读取项目根目录的 `.env`。

启动后端：

```bash
cd backend
python3 -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
uvicorn main:app --reload --port 8000
```

启动前端：

```bash
cd frontend
pnpm install
pnpm run dev
```

浏览器打开 `http://localhost:3000` 即可开始使用。

## 项目结构

- `frontend/`：前端页面、组件和前端代理
- `backend/`：后端接口、地点整理、路线生成和校验
- `database/`：数据库结构参考
- `examples/`：样例输入输出
- `docs/ARCHITECTURE.md`：更完整的架构说明

## 当前边界

- 当前版本以本地运行和完整规划闭环为重点
- 输入范围聚焦用户主动提供的文本资料
- 地图能力以高德地点链接为主
